# 🇰🇷 GitTrend Korea (깃트렌드 코리아) — 기획 제안서

> **"한국인이 오픈소스에 빠져들게 만드는 플랫폼"**

---

## 0. 프로젝트 한 줄 요약

GitHub에서 가장 핫한 레포지토리를 **실시간 한국어 번역**으로 보여주고, **유사 레포 추천** + **포크 쇼케이스 커뮤니티**를 통해 한국 개발자들이 오픈소스에 자연스럽게 참여하도록 만드는 웹 플랫폼.

---

## 1. 프로젝트 비전 & 문제 정의

### 1.1 풀고자 하는 문제

| 문제 | 현상 | 결과 |
|------|------|------|
| **영어 장벽** | GitHub 트렌딩의 모든 정보가 영어 | 한국 주니어 개발자의 진입 장벽 ↑ |
| **맥락 부재** | README를 읽어도 "이걸 왜 써야 하는지" 모름 | 좋은 도구를 놓침 |
| **참여 장벽** | Fork는 했지만 "어떻게 활용하는지" 사례 부족 | 오픈소스 기여 인구 정체 |
| **발견 어려움** | 비슷한 기능의 대안 라이브러리를 찾기 힘듦 | 기술 선택의 폭이 좁음 |

### 1.2 비전

- 🎯 **단기**: 한국 개발자가 매일 아침 확인하는 "오픈소스 뉴스 사이트"
- 🚀 **중기**: 포크 쇼케이스를 통한 한국 오픈소스 커뮤니티 활성화
- 🌏 **장기**: 한국발 오픈소스 프로젝트의 글로벌 진출 허브

---

## 2. 타겟 사용자

### Persona 1: 🧑‍🎓 주니어 개발자 "민수" (25세)
- 부트캠프 수료 후 취업 준비 중
- GitHub 트렌딩을 보고 싶지만 영어가 부담
- **핵심 니즈**: 한글 요약 + 실사용 예시

### Persona 2: 👩‍💻 중급 개발자 "서연" (30세)
- 실무 3년차, 새로운 기술 스택 탐색 중
- **핵심 니즈**: 유사 레포 비교 + 포크 쇼케이스

### Persona 3: 🧓 시니어 개발자 "재혁" (38세)
- 팀에 기술 도입을 결정하는 역할
- **핵심 니즈**: 주간 트렌드 요약 + 카테고리별 필터

## 3. 핵심 기능 (5대 모듈)

| # | 모듈 | 설명 | 우선순위 |
|---|------|------|----------|
| 1 | **트렌딩 대시보드** | GitHub 트렌딩 레포를 한국어로 보여주는 메인 화면 | MVP |
| 2 | **레포 한글 상세 페이지** | README 한글 번역 + AI 3줄 요약 | MVP |
| 3 | **유사 레포 추천** | GitHub Topics 기반 유사 레포 추천 | MVP |
| 4 | **포크 쇼케이스** | "나는 이 레포를 이렇게 활용했다" 커뮤니티 | Phase 2 |
| 5 | **주간 다이제스트** | 주간 트렌드 요약 뉴스레터 | Phase 3 |
